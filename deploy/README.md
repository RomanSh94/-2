# deploy/ — готовые артефакты для двух пунктов backlog

Эти файлы НЕ применяются автоматически и НЕ трогают действующий прод
(`/root/-2`). Это готовые к запуску скрипты/конфиги для двух задач.

## 1. Не-root деплой (Вариант A)
Для **нового** сервера: бот под отдельным пользователем `x20` в `/opt/x20`,
с systemd-hardening (NoNewPrivileges, ProtectSystem=strict).

- `x20-opt.service` — systemd-unit (User=x20, /opt/x20).
- `setup_opt.sh` — идемпотентный установщик с нуля.

```bash
bash deploy/setup_opt.sh        # под root на новом сервере
# затем: заполнить /opt/x20/.env, перенести x20.db, systemctl start x20
```

Перенос боевой `x20.db`: остановить старый инстанс (`systemctl stop x20`),
`scp x20.db x20.db-wal x20.db-shm` на новый в `/opt/x20/`, `chown x20:x20`,
запустить. **Один `getUpdates` на токен** — старый инстанс должен быть выключен.

> Действующий прод мигрировать «на горячую» не нужно и рискованно — это для
> момента переезда на новый сервер.

## 2. HTTPS для дашборда
Дашборд слушает только `127.0.0.1:8080`. Чтобы открыть наружу безопасно —
nginx-реверс-прокси + Let's Encrypt.

- `nginx-x20.conf` — конфиг реверс-прокси (подставь домен).
- `setup_https.sh` — ставит nginx+certbot, выпускает сертификат.

```bash
DOMAIN=dash.example.com EMAIL=you@example.com bash deploy/setup_https.sh
```

**Предусловия:** домен с A-записью на IP сервера, открытый порт 443 в Hetzner
firewall, и осознанное решение открыть дашборд наружу. До этого **SSH-туннель
(`ssh -L 8080:127.0.0.1:8080 root@SERVER`) — достаточно и безопаснее**: дашборд
защищён лишь одним паролем.
